# 高德地图 Key 配置说明

打开店铺「查看位置」时若出现 **Script error** 或控制台报错 **USERKEY_PLAT_NOMATCH**，说明当前使用的高德 Key 与平台/域名不匹配，需要在高德开放平台调整配置。

## 原因说明

- **USERKEY_PLAT_NOMATCH**：当前 Key 未开通「Web 端(JS API)」，或未把当前访问的域名加入白名单。
- 本地开发时访问地址一般为 `http://localhost:8080` 或 `http://127.0.0.1:8080`，需将该域名加入 Key 的白名单。

## 解决步骤

1. **登录高德开放平台**  
   打开 [https://lbs.amap.com](https://lbs.amap.com)，使用你的账号登录。

2. **进入应用与 Key 管理**  
   进入「控制台」→「应用管理」→ 找到使用 Key `0909b2b1157cec562f808a43c5d00ec8` 的应用（或创建新应用）。

3. **添加 / 检查 Key**  
   - 若该应用下还没有 Key：点击「添加 Key」。
   - 类型选择：**Web 端 (JS API)**。
   - 若已有 Key 但类型不是 Web 端：需要为该应用再添加一个「Web 端 (JS API)」的 Key，并在前端代码中替换为新 Key。

4. **设置 Key 白名单（重要）**  
   在 Key 的「设置」或「安全设置」中：
   - 将 **服务平台** 勾选为 **Web 端 (JS API)**（若可选）。
   - 在 **白名单** 或 **授权 IP/域名** 中至少添加：
     - `localhost`
     - `127.0.0.1`
   - 若部署到线上，还需添加你的正式域名（如 `www.example.com`）。

5. **保存并等待生效**  
   保存后，高德侧可能需数分钟生效。保存后刷新页面再试「查看位置」。

## 前端使用的 Key

当前项目中使用的高德 Key 在：

- `qd/src/components/ShopMapDialog.vue` 顶部常量 `AMAP_KEY`。

若你在控制台创建的是新的 Web 端 Key，请在该文件中把 `AMAP_KEY` 的值替换为新的 Key。

## 安全说明

- Key 不要提交到公开仓库；生产环境建议使用环境变量或后端下发。
- 在高德控制台可设置 Key 的「安全密钥」，并按高德文档在页面中配置，以降低 Key 被滥用的风险。
